fix-bar（固定バー）実装解説

トップページ右下に固定表示される「ページトップボタン」と「お問い合わせ」ボタンの仕組みを、 HTML・CSS・JavaScript の役割分担とともに図解します。

1. 全体像

fix-bar は次の 3つの役割 を持ちます。

役割 担当 内容
① 表示タイミング JavaScript MV（メインビジュアル）が画面外に出たら表示（is-visible）
② 縦位置の制御 JavaScript + CSS 通常は下から 20px。フッターが近づくと上に押し上げる
③ 見た目・レイアウト CSS（FLOCSS） 固定配置・ボタン配置・フェードイン

関連ファイル

wordpress/themes/.../template-parts/front-page/fix-bar.php ← HTML
src/assets/styles/layouts/_l-fix-bar.scss ← 位置（Layout）
src/assets/styles/projects/_p-fix-bar.scss ← 見た目（Project）
src/assets/styles/projects/_page-top.scss ← ページトップ丸ボタン
src/assets/js/_fix-bar.js ← 挙動（script.js から import）

2. HTML 構造（fix-bar.php）

<div class="l-fix-bar p-fix-bar js-fix-bar">
  <div class="p-page-top">
    <img src=".../arrow-t.svg" alt="Page Top">
  </div>
  <a href="/contact" class="l-btn c-btn c-btn--primary">お問い合わせ</a>
</div>

FLOCSS クラスの意味

  • l-fix-bar Layout … 画面上の固定位置（bottom / right / z-index）
  • p-fix-bar Project … このコンポーネント固有の見た目（flex・透明度）
  • js-fix-bar JavaScript … スクリプトのフック用（スタイルは原則当てない）
  • c-btn Component … ボタン共通スタイル

3. CSS の役割

3-1. Layout（_l-fix-bar.scss）— 座標の土台

.l-fix-bar {
  position: fixed;
  right: calc(20 * var(--to-rem));
  bottom: var(--fix-bar-bottom, calc(20 * var(--to-rem)));
  z-index: 850;
  transition: bottom 0.2s ease;
}
  • position: fixed … スクロールしても画面（viewport）に張り付く
  • --fix-bar-bottom … JS が書き換える CSS 変数。デフォルトは下から 20px 相当
  • transition: bottom … フッター回避で bottom が変わるとき、0.2 秒でなめらかに移動

3-2. Project（_p-fix-bar.scss）— 表示 / 非表示

.p-fix-bar {
  opacity: 0;
  pointer-events: none;

  &.is-visible {
    opacity: 1;
    pointer-events: auto;
  }
}

初期状態は透明でクリック不可。JS が is-visible を付けたときだけ表示されます。

4. 縦位置の計算（フッター回避）

bottom は「要素の下端が、画面の下端から何 px 上にあるか」を表します。

bottom = Math.max(20, window.innerHeight − footerTop + FOOTER_GAP)
※ FOOTER_GAP は現在 0（フッターにぴったり接する）

状態 A：フッターがまだ遠い（通常）

footerTop が画面の高さより大きい → 計算結果が 20 未満 → bottom = 20px のまま。

状態 B：フッターが近づいた

フッター上端が画面内に入る → bottom を大きくして fix-bar を上に押し上げ、下端がフッター上端に触れる位置に合わせる。

5. JavaScript の処理フロー（_fix-bar.js）

5-1. 起動タイミング

DOMContentLoaded
→
要素取得
.js-fix-bar
.p-mv
.p-footer
→
各種イベント登録
scroll / resize

5-2. 位置更新（スクロール時）

5-3. MV との表示切り替え（IntersectionObserver）

IntersectionObserver → .p-mv を監視

MV が画面内にある   → is-visible を外す（非表示）
MV が画面外に出た   → is-visible を付与（表示）+ 位置を再計算

5-4. ページトップクリック

.p-page-top クリック
  → e.preventDefault()
  → window.scrollTo({ top: 0, behavior: 'smooth' })

6. CSS 変数と JS の連携

JS は見た目を直接いじらず、CSS 変数だけを更新します。 実際の座標は SCSS 側の bottom: var(--fix-bar-bottom) が担当するため、 責務が分離されています（FLOCSS の考え方に沿った形）。

7. まとめ

一言でいうと： fix-bar は「MV を過ぎたら右下に出る固定 UI」で、スクロールに応じて bottom を動的に変え、フッターと重ならないようにしています。
  1. HTML … 構造と FLOCSS クラス（js- は JS 用フック）
  2. CSS Layout … position: fixed + CSS 変数で縦位置
  3. CSS Project … is-visible でフェード表示
  4. JS … スクロールで --fix-bar-bottom 更新、MV 監視で表示切替、クリックでトップへ
